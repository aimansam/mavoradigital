# Bahasa Melayu Landing Page Copy Map

## Status

Fasa 1 and Fasa 2 are ready for implementation.

This document maps the current English landing page copy to Bahasa Melayu copy for the first `/bm/` version. The goal is not a literal translation. The BM version should sound natural for Malaysian business owners and keep the flow focused on package value, trust, and WhatsApp project requests.

## Voice Rules

- Use `anda` when speaking to the visitor.
- Use `bisnes`, `website`, `landing page`, `WhatsApp`, `domain`, `hosting`, `SEO`, and `portfolio` where these sound more natural than formal Malay alternatives.
- Avoid making the form feel like a formal quotation system. Use `permintaan projek`, `butiran projek`, or `ringkasan projek`.
- Keep pricing language direct: `Bermula RM800`, `RM1,200`, `3-5 hari bekerja`.
- CTA language should be action-focused: `Hantar Permintaan Projek`, `Lihat Servis`, `Lihat Semua Demo`, `Hantar ke WhatsApp`.

## Implementation Notes For Fasa 3

- Extract page copy into a typed content object before creating `/bm/`.
- Keep arrays like packages, website types, form options, and status strings in the content object.
- Pass copy into the page script through `define:vars` or `data-*` attributes for form states and WhatsApp message labels.
- Translate `ProcessCardStack.jsx` or allow it to accept localized process items as props.
- Add a small language switcher with `English` and `BM`; keep it secondary to the main CTA.
- Use `src/data/site/` for TypeScript copy data. `src/content/` is reserved for Astro content collections and will create collection warnings for this use case.
